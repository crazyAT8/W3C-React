React ES6 Arrow Functions

    Arrow Functions:
  • allows us to write shorter function suntax.
  • if the function has only one statement,
    and the statement returns a value,
    you can remove the brackets and the return keyword
    works only if the function has only one statement.
  • parameters are passed inside the parentheses
    and if there is only one then you don't need parentheses

What About this?

  • the handling of this is also different in arrow functions compared to regular functions.
  • with arrow functions there is no binding of this
  • in regular functions the this keyword
    represented the object that called the function,
    which could be the window, the document, a button or whatever
  • with arrow functions, the this keyword
    always represents the object that defined the arrow function.
  • Remember: this keyword represents different objects depending on how the function was called
